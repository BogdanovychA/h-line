# 📝 H-Line (Hotline App)

[![DOI](https://zenodo.org/badge/1211130746.svg)](https://doi.org/10.5281/zenodo.19590690) ![GitHub License](https://img.shields.io/github/license/BogdanovychA/h-line?logoColor=grey&color=blue)

Застосунок для швидкої фіксації та надсилання звернень громадян через електронну пошту. Побудований на базі **Flet** (Python), підтримує генерацію документів (DOCX, MD, HTML) та автоматичне відправлення через SMTP.

## 🚀 Як запустити локально

Для роботи рекомендується використовувати менеджер пакетів [uv](https://github.com/astral-sh/uv).

### 1. Підготовка налаштувань
Створіть файл `.env` у папці `src/assets/` (можна скопіювати з прикладу):
```bash
cp src/assets/.env.example src/assets/.env
```
Відредагуйте `.env`, вказавши ваші налаштування SMTP (Gmail, Outlook тощо) та email отримувача.

### 2. Запуск через uv
Встановіть залежності та запустіть:

  ```bash
  uv sync
  ```
* **Як desktop-застосунок:**
  ```bash
  uv run flet run
  ```
* **Як вебзастосунок:**
  ```bash
  uv run flet run --web
  ```

## 🐳 Запуск через Docker

Це найкращий варіант для розгортання застосунку на сервері.

### 1. Збірка та запуск
Використовуйте `docker-compose` для автоматичного налаштування:

```bash
docker-compose up -d
```

### 2. Налаштування контейнера
* **Порт:** Застосунок буде доступний за адресою `http://localhost:8081`.
* **Змінні оточення:** Docker використовує файл `src/assets/.env`.
* **Дані:**
    * Вихідні файли зберігаються в `./docker_data/output`.
    * Шаблони необхідно розмістити в `./docker_data/templates`.

## 🛠 Можливості та архітектура
Проєкт спроектований за принципами **SOLID** та **DRY**, що забезпечує високу гнучкість:

- ✅ **Генерація документів:** Автоматичне заповнення шаблонів звернень (DOCX, MD, HTML).
- ✅ **Гнучкий UI:** Підтримка розширених полів (категорія заявника, соціальний стан) згідно з офіційними класифікаторами.
- ✅ **SMTP інтеграція:** Підтримка SSL/TLS через абстрактні поштові протоколи.
- ✅ **Модульність (Registry Pattern):** Легке додавання нових форматів генерації або способів відправки без зміни ядра системи.
- ✅ **Аналітика:** Вбудована підтримка Google Analytics (Measurement API).
- ✅ **Безпека:** Чітке розділення конфігурації через змінні оточення.

## 👨‍💻 Для розробників (як розширити?)

Завдяки впровадженим патернам проектування, ви можете легко додавати новий функціонал:

1. **Новий формат генерації:** Створіть клас-спадкоємець `BaseTemplateManager` та зареєструйте його в `src/ui/main.py`:
   ```python
   GlobalGenerator.register(FileType.PDF, MyPdfManager)
   ```
2. **Новий поштовий протокол:** Створіть реалізацію `SMTPSenderBase` та зареєструйте її:
   ```python
   EmailManager.register(SMTPProtokol.CUSTOM, MyCustomSender)
   ```
